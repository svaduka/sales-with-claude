# Opportunity Analysis: Quorum Data & AI Platform (op1)

## Opportunity Name
**Quorum Data & AI Platform (QDAI) – Senior Data Platform DevOps Engineer with EPAM**

---

## Short Summary
Quorum, an energy software company, is building a unified Data & AI Platform (QDAI) to transform their fragmented data landscape into a Knowledge Graph that powers Agentic AI. They are seeking a strategic implementation partner (EPAM) to deliver an Azure + Databricks-based data platform with expertise in architecture, DevOps, and knowledge transfer. The opportunity includes both a consulting engagement and a specific Senior Data Platform DevOps Engineer role.

---

## Key Business Needs

| Business Need | Description |
|---|---|
| **Data Silo Elimination** | Transform fragmented systems across 3 product segments (North America O&T, Upstream Planning, International O&T) into a unified platform |
| **Relational Intelligence** | Build a Knowledge Graph linking land, production, and accounting data across the energy lifecycle |
| **Agentic AI Foundation** | Enable AI agents to perform complex, multi-step tasks autonomously across product boundaries |
| **SaaS Modernization** | Long-term transition to SaaS-first platform while delivering immediate intelligence capabilities |
| **Cost Optimization** | Deliver cost-competitive, reliable, and secure data platform capabilities |
| **Standardized Integration** | Provide seamless data integration between Quorum products and customer enterprise ecosystems |
| **Self-Service Analytics** | Enable end users to access and work with data instantly, improving productivity |

---

## Key Technical Needs

| Technical Area | Requirements |
|---|---|
| **Platform Architecture** | Scalable, secure, cost-effective Azure + Databricks + Microsoft Fabric architecture |
| **Core Technology Stack** | Databricks (Delta Lake, Unity Catalog, DLT), Azure Data Factory, Azure Synapse, Azure Data Lake Storage Gen 2, Azure SQL Database |
| **Data Architecture** | Medallion Architecture (Bronze, Silver, Gold) with MDM integration in Silver layer |
| **Data Governance** | Unity Catalog for centralized governance, Azure Purview for catalog & lineage, data quality enforcement |
| **DevOps & Automation** | Infrastructure as Code (Terraform, Ansible), CI/CD pipelines (Azure DevOps), scripting (Python, PowerShell, Azure CLI) |
| **Pipeline Development** | Fully automated, modular, resilient ETL pipelines using Azure data services |
| **Security Model** | Data encryption, network security (Private Links), IAM, RBAC/ABAC access controls |
| **Knowledge Graph** | Unified Semantic Layer / Ontology (TBD - under investigation) to power Agentic AI |
| **Multi-Region & Multi-Tenancy** | Global accessibility with isolated, secure customer environments |
| **Master Data Management** | Single source of truth for Meters, Locations, Products across fragmented source systems (TIPS, QPTM, eONE, FLOWCAL) |
